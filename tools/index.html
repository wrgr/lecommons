Platforms, systems & software

What platforms and systems do learning engineers build on?

Tools (1)

Tool Knowledge RepresentationIntelligent Tutoring & Adaptive Systems

CTAT — Cognitive Tutor Authoring Tools

Carnegie Mellon University

Authoring environment for building intelligent tutoring systems with cognitive models. Allows learning engineers to encode expert knowledge as production rules and knowledge components, then test those models against student interaction data.

DataShop / LearnSphere

Carnegie Mellon University

Open repository of educational datasets and analysis tools. Hosts interaction data from intelligent tutoring systems and supports Learning Factors Analysis, knowledge component modeling, and educational data mining at scale.

Platform ICICLE resources Intelligent Tutoring & Adaptive SystemsLearning Infrastructure & Platforms

GIFT — Generalized Intelligent Framework for Tutoring

US Army Research Laboratory

Open-source framework for authoring, delivering, and assessing intelligent tutoring systems. Army-funded; widely used in defense and adjacent workforce training contexts.

Platform Standards Learning Infrastructure & PlatformsStandards, Credentialing & Community

xAPI — Experience API specification

IEEE LTSC / ADL

The de facto standard for learning-event instrumentation. Every LE platform conversation eventually comes back to xAPI. Runs through the IEEE LTSC, where much of ICICLE's standards work also happens.

Yet Analytics — Learning Engineering Tools

Yet Analytics

Commercial learning-engineering platform spanning xAPI data design, instrumentation, and evaluation. One of the few vendors building tools explicitly for the LE workflow rather than generic ed-tech analytics.
